# Proto-contract compliance

The gRPC contract is schema-first.

```text
official Buf module -> generated Python packages -> Zorn service implementation
```

The project must not define parallel `.proto` files for Lattice compatibility. The only accepted contract source is:

```text
buf.build/anduril/lattice-sdk
```

## Locked Python artifacts

The pinned Python artifacts are generated by Buf from the public Lattice module:

```text
anduril-lattice-sdk-grpc-python==1.80.0.1.20260515215502+ed34febdefc1
anduril-lattice-sdk-protocolbuffers-python==34.1.0.1.20260515215502+ed34febdefc1
anduril-lattice-sdk-protocolbuffers-pyi==34.1.0.1.20260515215502+ed34febdefc1
```

Install them with:

```bash
./scripts/install_grpc_deps.sh
```

That script installs `requirements/grpc-runtime.txt` from PyPI and installs the generated Lattice artifacts from `requirements/lattice-buf-generated.txt`, which points only at Buf's generated Python package index. `requirements/grpc-buf.txt` is kept as a convenience include file for both requirement sets.

## Runtime verification

At gRPC server startup, Zorn introspects the installed generated descriptors and verifies that these public services and stream cardinalities exist:

```text
anduril.entitymanager.v1.EntityManagerAPI
  unary         PublishEntity
  client stream PublishEntities
  unary         GetEntity
  unary         OverrideEntity
  unary         RemoveEntityOverride
  server stream StreamEntityComponents

anduril.taskmanager.v1.TaskManagerAPI
  unary         CreateTask
  unary         GetTask
  unary         QueryTasks
  unary         UpdateStatus
  unary         CancelTask
  server stream ListenAsAgent
  server stream ListenForManualControlFrames
  server stream StreamTasks
```

Generate a local report with:

```bash
python scripts/proto_contract_report.py --assert --pretty --output docs/generated/proto-contract-report.json
```

If an upstream Buf package changes service names, method names, request/response message full names, or streaming cardinality, the server fails at startup instead of silently serving a drifted mock.

## Behavioral compliance vs proto compliance

Proto compliance means the wire schema, service names, method names, and stream cardinality come from the official public protos. Behavioral compliance is tracked separately and still needs contract tests against official sample apps.

Current behavior hardening in this slice:

```text
- gRPC CreateTask defaults to STATUS_SENT, matching the gRPC reference.
- UpdateStatus accepts the public status_update envelope.
- ListenAsAgent and StreamEntityComponents now honor heartbeat_interval_ms / heartbeat_period_millis zero as no heartbeat.
- StreamEntityComponents rejects include_all_components plus components_to_include together.
- ListenAsAgent supports agent_selector.entity_ids.entity_ids for basic routing.
- Response serialization now filters payloads through protobuf descriptors before parsing.
```
