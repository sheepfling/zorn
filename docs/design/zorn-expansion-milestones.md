# Zorn Expansion Milestones

## Z0: Core Compatibility Guardrails

Keep the API boundary strict. Tests must catch invented third-party-facing API
in ecosystem manifests and route docs.

## Z1: Entity / Task / Object Hardening

Improve lifecycle behavior, canonical normalization, object metadata fidelity,
and task state transitions behind existing Lattice-shaped surfaces.

## Z2: SDK Compatibility Harness

Expand official SDK and sample-app conformance with clear supported, partial,
unsupported, and blocked results.

## Z3: Sandbox And Adapter Foundations

Make adapters such as DIS and AIS plugin-like clients of existing Entity, Task,
Object, Auth, and stream surfaces.

## Z4: Developer Console And C2 MVP

Build local product apps that consume the data plane without defining new
integration API.

## Z5: Mesh Simulator V1

Simulate nodes, link quality, offline queues, and object availability as local
scenario behavior over existing state.

## Z6: Partner / Integration System

Add package manifests, scorecards, fixture bundles, and conformance reports for
apps, adapters, agents, and data services.

## Z7: Multi-Agent Scenario Lab

Model taskable agents, mission graphs, TaskCatalog matching, cancellation, and
human-supervision gates.

## Z8: Domain Plugin Architecture

Support domain packs for radar, EO/IR, C-UAS, air autonomy, maritime/undersea,
tactical compute, and space-domain awareness.

## Z9: Advanced Domain Scenario Packs

Add deterministic scenario packs with reference data, expected timelines, and
cross-surface certification checks.
