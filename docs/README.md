# Docs

Canonical repo notes live here.

- `plans/` is the backlog and execution packet area.
- `plans/zorn-next-milestones-execution-plan.md` defines the Z3-Z12 execution plan.
- `reference/ui/` preserves the UI reference registry that informs the Zorn COP/debug UI.
- `reference/lattice-ecosystem-*.json` records ecosystem lanes and requirements without expanding the public API.
- `reference/zorn-next-*.json` records the next milestone execution plan.
- `reference/zorn-showcase-*.json` records showcase milestones and planned scenario coverage.
- `design/alpha-readiness-roadmap.md` maps the technical roadmap to Alpha 1/2/3 release gates.
- `design/alpha1-gap-closure-plan.md` captures the external-evaluator corrective work needed to turn Alpha1.1 into a stricter Alpha 1 without expanding the public surface.
- `design/zorn-alpha-expansion-plan.md` captures the downstream Alpha 2/3 plan for the tactical sandbox and UI layers above the strict surrogate core.
- `design/zorn-configuration.md` documents the startup configuration surface, including the optional TOML profile and the broader non-auth knobs.
- `design/strict-startup-contract.md` captures the fail-fast startup profile used to keep the surrogate aligned with the public Lattice boundary.
- `design/zorn-lattice-ecosystem-architecture.md` defines module boundaries for SDK, C2, autonomy, mesh, partner, adapter, and domain work.
- `design/zorn-full-feature-showcase-roadmap.md` defines the Z0-Z12 showcase plan.
- `design/zorn-product-boundaries.md` defines the separation between the compatibility data plane and Zorn product apps.
- `design/zorn-cop-debug-ui.md` defines the staged plan for the first usable UI.
- Add longer workflow or domain notes here as the surrogate grows.
